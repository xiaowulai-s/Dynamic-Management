## 1. 概述

本规范定义设备信息动态管理系统前端的视觉与交互统一标准，覆盖表单对齐、输入控件尺寸、布局间距等核心视觉一致性规则。所有新页面与组件必须遵循本规范，既有页面在迭代时须对齐至本规范。

全局样式入口：`frontend/src/styles/index.css`

## 2. 表单对齐规范

### 2.1 问题背景

未规范化前，表单存在三类典型不对齐问题：

| 问题类型 | 根因 | 表现 |
|---------|------|------|
| 宽度不齐 | `el-input` 默认宽度 auto，`el-date-picker`/`el-select` 加了 `form-full`（100%） | 同一行左右两列输入框宽度不一致，文字终止位置不在同一竖线 |
| 高度不齐 | `.el-input__wrapper` 用 `padding: 8px 16px` 撑到 46px，`el-date-picker`（36px）、`el-select__wrapper`（32px）不受影响 | 跨行文字基线不在同一水平线 |
| 标签错位 | `.el-form-item__content` 未强制垂直居中 | label 文字与 input 文字垂直方向错位 |

### 2.2 对齐规则

所有表单输入控件必须遵循以下规则：

| 规则 | 要求 | 说明 |
|------|------|------|
| 宽度统一 | `width: 100%` | 同行左右两列、同表单不同行的输入框右边缘对齐 |
| 高度统一 | `height: 36px` | 所有单行输入控件高度一致，文字基线在同一水平线 |
| 垂直居中 | `align-items: center` | label 与 content 垂直居中，避免文字错位 |
| textarea 例外 | `min-height: 36px` | 多行输入不强制高度，保留扩展能力 |

### 2.3 覆盖控件类型

统一规则适用于 `.el-form-item__content` 内的以下控件：
- `el-input`（单行文本输入）
- `el-select` / `el-select__wrapper`（下拉选择，含新版 Element Plus）
- `el-date-editor`（日期选择器）
- `el-input-number`（数字输入）
- `el-cascader`（级联选择）
- `el-time-picker` / `el-time-select`（时间选择）
- `el-autocomplete`（自动补全）
- `el-color-picker`（颜色选择）
- `el-slider`（滑块）
- `el-textarea`（多行文本，仅 min-height）

### 2.4 实现代码

位于 `frontend/src/styles/index.css` 的 `/* Form */` 区块：

```css
/* 表单对齐规范 */
.el-form-item__content {
  display: flex;
  align-items: center;
  flex-wrap: wrap;
  line-height: var(--leading-normal);
}

.el-form-item__content .el-input,
.el-form-item__content .el-select,
.el-form-item__content .el-date-editor.el-input,
.el-form-item__content .el-date-editor.el-input__wrapper,
.el-form-item__content .el-input-number,
.el-form-item__content .el-cascader,
.el-form-item__content .el-time-picker,
.el-form-item__content .el-time-select,
.el-form-item__content .el-autocomplete,
.el-form-item__content .el-color-picker,
.el-form-item__content .el-slider {
  width: 100%;
}

.el-form-item__content .el-input__wrapper,
.el-form-item__content .el-date-editor.el-input__wrapper {
  height: 36px;
  box-sizing: border-box;
}

.el-form-item__content .el-select__wrapper {
  height: 36px;
  min-height: 36px;
  box-sizing: border-box;
}

.el-form-item__content .el-textarea__inner {
  min-height: 36px;
  box-sizing: border-box;
}
```

`.el-input__wrapper` 的 padding 由 `8px 16px` 收为 `4px 12px`，以保证内容高度不超过 36px。

## 3. 输入控件尺寸规范

### 3.1 标准尺寸

| 控件类型 | 高度 | 宽度 | 内边距 |
|---------|------|------|--------|
| 单行输入 `el-input` | 36px | 100% | `4px 12px` |
| 下拉选择 `el-select` | 36px | 100% | 跟随 Element Plus 默认 |
| 日期选择 `el-date-picker` | 36px | 100% | 跟随 Element Plus 默认 |
| 多行输入 `el-textarea` | ≥36px（自适应） | 100% | 跟随 Element Plus 默认 |

### 3.2 圆角与边框

- 圆角：`var(--radius-sm)`（与设计系统一致）
- 背景色：`var(--surface-input)`
- 过渡：`border-color var(--transition-fast), box-shadow var(--transition-fast)`

**边框三态规范**（v1.3 新增，修复"未聚焦时无边框"问题）：

Element Plus 默认用 `box-shadow: 0 0 0 1px <color> inset` 渲染输入框边框，而非 `border` 属性。若全局样式设 `box-shadow: none` 但只设 `border-color` 不设 `border`，会导致未聚焦时无边框。本系统改用真实 `border`，三态如下：

| 状态 | 边框 | 阴影 | 适用控件 |
|------|------|------|---------|
| 默认（未聚焦） | `1px solid var(--border-default)` | none | input / select / textarea |
| hover | `1px solid var(--border-hover)` | none | input / select / textarea |
| focus | `1px solid var(--border-focus)` | `0 0 0 3px rgba(37,99,235,0.12)` | input / select / textarea |

覆盖的选择器：
- `.el-input__wrapper`（单行输入）
- `.el-select__wrapper`（下拉选择，Element Plus 新版）
- `.el-textarea__inner`（多行输入）

## 4. 布局间距规范

### 4.1 表单项间距

- 表单项垂直间距：`var(--space-5)`（`margin-bottom`）
- label 与 content 间距：`var(--space-2)`（`padding-bottom`）

### 4.2 对话框表单布局

对话框内的表单推荐使用双列布局（`el-row` + `el-col`），同行左右两列的 `el-form-item` 共享同一 `gutter`，由全局对齐规则保证视觉一致。

## 5. 表格布局与对齐规范

### 5.1 问题背景

表格普遍存在两个问题：
1. **列宽固定不自适应** — 所有列用 `width="150"` 固定值，内容短的列浪费空间，导致大量空白、排列不均
2. **操作按钮换行** — 操作列 `width` 固定但按钮无容器包裹、无 nowrap 控制，按钮总宽度接近列宽时换行，显示不整齐

### 5.2 列宽规则

所有数据列统一使用 `min-width` 而非 `width`，让 Element Plus 按内容自适应分配剩余空间：

| 列类型 | min-width 参考 | 说明 |
|--------|:---:|------|
| 编号/ID | 90-120 | 短标识符 |
| 名称 | 140-160 | 主标识，可设 `show-overflow-tooltip` |
| 型号/位置 | 100-120 | 中等长度文本 |
| 状态/标签 | 80-110 | 含 el-tag 或 badge |
| 时间 | 150-180 | 日期时间字符串 |
| 操作 | 140-220 | 按钮组，按按钮数量定 |

**禁止**：使用 `width` 固定列宽（会导致内容短的列浪费空间、内容长的列被截断）。

### 5.3 对齐规则（按列类型区分）

用 Element Plus 原生 `align` 属性控制，表头与单元格同步对齐：

| 列类型 | align | 说明 |
|--------|:---:|------|
| 文本列（名称/编号/型号/位置/描述/提交人） | `left`（默认，可省略） | 长文本左对齐便于阅读 |
| 状态/标签列（状态/生命周期/日志类型/角色） | `center` | 标签居中视觉平衡 |
| 时间列（创建时间/提交时间） | `center` | 时间字符串居中 |
| 数字列（设备数/计数） | `center` | 数字居中 |
| 操作列（查看/编辑/删除） | `center` | 按钮组居中 |

**示例**：

```vue
<!-- 文本列：默认左对齐 -->
<el-table-column prop="name" label="设备名称" min-width="140" show-overflow-tooltip />

<!-- 状态/标签列：居中 -->
<el-table-column prop="status" label="状态" min-width="80" align="center">
  <template #default="{ row }">
    <el-tag :type="getStatusType(row.status)">{{ getStatusLabel(row.status) }}</el-tag>
  </template>
</el-table-column>

<!-- 操作列：居中 + table-actions 容器 -->
<el-table-column label="操作" min-width="180" fixed="right" align="center">
  <template #default="{ row }">
    <div class="table-actions">
      <el-button type="primary" size="small" @click="handleView(row)">查看</el-button>
      <el-button type="warning" size="small" @click="handleEdit(row)">编辑</el-button>
      <el-button type="danger" size="small" @click="handleDelete(row)">删除</el-button>
    </div>
  </template>
</el-table-column>
```

### 5.4 操作列按钮规则

操作按钮必须用 `.table-actions` 容器包裹，强制一行显示：

```css
/* 全局样式（styles/index.css） */
.el-table .table-actions {
  display: flex;
  flex-wrap: nowrap;
  gap: 4px;
  align-items: center;
}

.el-table .table-actions .el-button {
  margin-left: 0;
  flex-shrink: 0;
}
```

**规则**：
- 所有操作按钮（查看/编辑/删除/审批等）必须放在 `<div class="table-actions">` 内
- `flex-wrap: nowrap` 确保不换行
- `flex-shrink: 0` 防止按钮被压缩
- `gap: 4px` 统一按钮间距

## 8. 部署与二次开发流程

### 7.1 架构与端口约定

| 组件 | 容器/进程 | 端口 | 用途 |
|------|----------|:---:|------|
| Nginx（生产前端） | `equipment-nginx` | **9000** | 唯一对外访问入口，托管 `frontend/dist` + 反代 `/api`、`/uploads` |
| FastAPI 后端 | `equipment-backend` | 8000 | API 服务，仅容器内网与本机调试可访问 |
| PostgreSQL | `equipment-postgres` | 5432 | 数据库 |
| Redis | `equipment-redis` | 6379 | 缓存与消息队列 |
| Vite Dev Server | 本地 node 进程 | 7000 | **仅开发调试用**，默认不启动 |

**端口策略**：所有用户统一通过 `http://<服务器IP>:9000` 访问。7000 端口的开发服务器仅供本地热更新调试，不对外暴露，日常运行不需要启动。

### 8.2 三个一键脚本

脚本均位于项目根目录 `d:\Demo\Dynamic-Management\`，双击即可执行：

| 脚本 | 用途 | 使用场景 |
|------|------|---------|
| `启动系统.bat` | 启动全套 Docker 服务并打开浏览器 | 每次开机后双击 |
| `更新前端.bat` | 重新构建 `frontend/dist` 并重启 nginx | 改完前端代码后双击 |
| `停止服务.bat` | 停止所有 Docker 容器 | 不需要服务时双击 |

脚本已做编码处理（GB2312 + 全英文提示），兼容 Windows cmd 默认环境。

### 8.3 日常使用流程（运维人员）

```
开机
  ↓
Docker Desktop 自动启动（设置里勾选 "Start when you sign in"）
  ↓
双击 启动系统.bat
  ↓
脚本自动：检查 Docker → 启动 4 容器 → 等待就绪 → 打开浏览器
  ↓
全员访问 http://<服务器IP>:9000
  ↓
不需要时双击 停止服务.bat
```

### 8.4 二次开发流程（开发者）

#### 8.4.1 开发调试（带热更新）

```bash
# 1. 启动后端服务（Docker）
docker compose up -d postgres redis backend

# 2. 启动前端开发服务器（本地，带热更新）
cd frontend
npx vite --host 0.0.0.0 --port 7000

# 3. 浏览器访问 http://127.0.0.1:7000
#    改前端代码自动生效，改后端代码后端容器自动 reload
```

开发时用 7000 端口（热更新），不影响 9000 端口的生产服务。

#### 8.4.2 改前端代码 → 发布到生产

改完 `frontend/src/` 下的任何文件（.vue / .ts / .css）后：

```
双击 更新前端.bat
  ↓
脚本自动执行：
  1. 检查 Node.js
  2. npm install（首次或依赖变更时）
  3. npx vite build  → 生成 frontend/dist
  4. docker restart equipment-nginx  → nginx 加载新 dist
  5. 轮询验证 http://127.0.0.1:9000 返回 200
  6. 打印访问地址
  ↓
浏览器按 Ctrl+F5 强制刷新（清缓存）
```

**验证方法**：改 `frontend/index.html` 的 `<title>` 加版本标记，跑完脚本后访问 9000 确认标题变化（已在 2026-07-19 验证通过，v1.3 标记生效）。

#### 7.4.3 改后端代码 → 发布到生产

后端代码在 `backend/app/` 下。由于 `docker-compose.yml` 挂载了 `./backend:/app` 且启动命令带 `--reload`，改后端 Python 代码后容器会自动热加载。

如需手动重启：

```bash
docker restart equipment-backend
```

#### 8.4.4 改数据库结构

数据库迁移流程：

```bash
# 1. 编写迁移 SQL 或修改 backend/app/models/ 下的模型
# 2. 直接在 PostgreSQL 容器执行迁移
docker exec -it equipment-postgres psql -U postgres -d equipment_db -c "ALTER TABLE ..."

# 3. 如改了模型，重启后端
docker restart equipment-backend
```

> 当前已执行的迁移：`equipment.created_by`、`logs.operator_id`、`feedbacks.user_id` 由 `NOT NULL` 改为可空（支持删除用户时置空引用）。

### 8.5 二次开发注意事项

| 场景 | 注意事项 |
|------|---------|
| 新增前端页面 | 在 `frontend/src/views/` 创建 .vue 文件，在 `frontend/src/router/index.ts` 注册路由，菜单项在 `Layout.vue` 添加 |
| 新增后端接口 | 在 `backend/app/routers/` 对应文件添加路由，权限校验参考 `docs/权限管理规范.md` 第 4 节 |
| 新增表单 | 无需手动加 `form-full` 类，全局样式已统一宽度/高度（见本文件第 2 节） |
| 新增控件类型 | 在 `frontend/src/styles/index.css` 的表单对齐区块补充选择器 |
| 改权限规则 | 同步更新 `docs/权限管理规范.md`，按该文档第 9 节做三账号回归验证 |
| 改样式系统 | 同步更新本文件第 3、4 节 |
| 提交代码前 | 跑 `更新前端.bat` 确认构建无错，跑三账号验证确认权限正常 |

### 8.6 故障排查

| 现象 | 排查步骤 |
|------|---------|
| 双击启动脚本提示 Docker 未运行 | 启动 Docker Desktop，等待右下角图标变绿后重试 |
| 9000 端口访问 502/504 | `docker compose ps` 查 nginx/backend 状态，`docker compose logs nginx` 查错误日志 |
| 更新前端后页面没变化 | 浏览器 Ctrl+F5 强制刷新；或 `docker restart equipment-nginx` 重启 |
| 构建失败 `vite build` 报错 | 检查 `frontend/src/` 下是否有语法错误、import 是否齐全 |
| 8000 端口后端不通 | `docker logs equipment-backend --tail 50` 查启动日志 |
| 端口被占用 | `netsh interface ipv4 show excludedportrange protocol=tcp` 查 Windows 保留端口，改用其他端口 |

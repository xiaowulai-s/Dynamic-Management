# 系统迁移部署指南

将设备信息动态管理系统迁移到另一台 Windows 服务器（含全部数据）的完整操作指南。

## 适用场景

- 当前服务器 → 新 Windows 服务器
- Docker 部署方式
- 全量数据迁移（数据库 + 上传文件 + 系统配置）
- 保持端口不变（前端 9000 / 后端 8000）

## 迁移流程总览

```
当前服务器                          新服务器
┌─────────────────┐                ┌─────────────────┐
│ 1. 运行         │   拷贝迁移包   │ 3. 安装 Docker  │
│    迁移-导出.ps1│ ────────────> │ 4. 运行         │
│ 2. 生成迁移包   │   U盘/网络     │    迁移-导入.ps1│
└─────────────────┘                │ 5. 访问 :9000   │
                                    └─────────────────┘
```

整个过程分为三大步骤：导出 → 传输 → 导入。

---

## 第一阶段：在当前服务器导出

### 1.1 前置检查

- Docker Desktop 已启动（系统托盘图标为绿色）
- 系统正在运行（`equipment-postgres` 等容器状态为 running）
- 磁盘剩余空间 > 500 MB

### 1.2 运行导出脚本

在项目根目录（`d:\Demo\Dynamic-Management`）右键点击 `迁移-导出.ps1`，选择"用 PowerShell 运行"。

脚本会自动完成以下工作：

| 步骤 | 操作 | 产物 |
|------|------|------|
| 1/5 | 创建导出目录 | `迁移包\` |
| 2/5 | pg_dump 导出数据库 | `data\equipment_db.dump` |
| 3/5 | 打包 uploads 上传文件 | `data\uploads.zip` |
| 4/5 | 复制代码与配置 | backend / frontend / nginx 等 |
| 5/5 | 生成说明文档 | `迁移说明.md` |

### 1.3 导出产物说明

导出完成后在项目根目录生成 `迁移包` 文件夹（约 6-10 MB），结构如下：

```
迁移包/
├── data/
│   ├── equipment_db.dump      # 数据库备份（PostgreSQL 自定义格式）
│   └── uploads.zip            # 上传文件压缩包
├── backend/                   # 后端代码 + Dockerfile
├── frontend/
│   ├── dist/                  # 前端构建产物（新服务器无需装 Node.js）
│   └── src/                   # 前端源码（供后续修改）
├── docker-compose.yml         # 服务编排配置
├── nginx.prod.conf            # Nginx 配置
├── 迁移-导入.ps1              # 新服务器部署脚本
├── 启动系统.bat               # 日常启动脚本
├── 更新前端.bat               # 前端重新构建脚本
├── 停止服务.bat               # 停止服务脚本
└── 迁移说明.md                # 操作说明
```

### 1.4 数据备份内容

数据库备份包含全量数据：

- 用户账号与角色（root、admin_test、user_test 等）
- 设备信息与设备类型
- 全部 8 类日志记录（安装、维修、报废、巡检、保养、故障、配件、校准）
- 审批记录与审批配置
- 客户信息
- 用户反馈
- 操作审计日志
- 系统配置（站点标题、预警天数、保养周期等）
- 通知消息

## 第二阶段：传输迁移包

将整个 `迁移包` 文件夹拷贝到新服务器。推荐方式：

| 方式 | 适用场景 | 备注 |
|------|----------|------|
| U盘/移动硬盘 | 内网环境 | 最稳定，无网络依赖 |
| 局域网共享文件夹 | 同一局域网 | 速度快，需开启文件共享 |
| 远程桌面文件传输 | 跨网络 | 直接拖拽到 RDP 窗口 |
| 网盘/邮件 | 跨网络小文件 | 迁移包约 6-10 MB，可压缩后传输 |

**建议**：在新服务器解压到 `D:\Dynamic-Management`（路径不要含中文或空格）。

---

## 第三阶段：在新服务器部署

### 3.1 安装 Docker Desktop

1. 下载 Docker Desktop for Windows：https://www.docker.com/products/docker-desktop/
2. 双击安装，按默认选项完成
3. 启动 Docker Desktop，等待系统托盘图标变为绿色（运行中）
4. 首次启动如提示 WSL 2，按提示安装并重启

**系统要求**：
- Windows 10/11 64 位（专业版/企业版/家庭版均可）
- 至少 4 GB 内存
- BIOS 开启虚拟化

### 3.2 运行导入脚本

在新服务器的迁移包根目录（如 `D:\Dynamic-Management`）右键点击 `迁移-导入.ps1`，选择"用 PowerShell 运行"。

脚本会自动完成以下工作：

| 步骤 | 操作 | 耗时 |
|------|------|------|
| 1/7 | 检查项目文件完整性 | < 1s |
| 2/7 | 构建镜像并启动服务 | 5-15 分钟（首次需拉取镜像） |
| 3/7 | 等待 PostgreSQL 就绪 | 10-30s |
| 4/7 | 导入数据库数据 | 5-30s |
| 5/7 | 还原上传文件 | < 10s |
| 6/7 | 检查前端构建产物 | < 1s |
| 7/7 | 重启服务使数据生效 | 5-10s |

首次部署需要从 Docker Hub 拉取以下镜像：
- `postgres:15` （数据库）
- `redis:7` （缓存/消息队列）
- `nginx:alpine` （前端服务）
- `python:3.11-slim` （后端，本地构建）
- `node:18-alpine` （前端构建，仅首次）

### 3.3 验证部署

脚本完成后会显示访问地址（新服务器的局域网 IP）。用浏览器访问：

```
http://新服务器IP:9000
```

**首次访问请用 Ctrl+F5 强制刷新**，避免浏览器缓存旧页面。

### 3.4 默认账号

数据迁移后账号保持不变：

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 超级管理员 | root | root123 |
| 管理员 | admin_test | 123456 |
| 普通用户 | user_test | 123456 |

登录后请前往"个人资料"修改默认密码。

## 常见问题

### Q1：导入脚本卡在"构建镜像"步骤很久

首次拉取 Docker 镜像需要从 Docker Hub 下载，国内网络可能较慢。如超过 30 分钟无进展：
1. 配置 Docker 镜像加速器（阿里云/腾讯云加速地址）
2. 或在能联网的机器上先 `docker pull` 好镜像再打包传输

### Q2：导入后访问页面显示空白

1. 检查浏览器是否缓存了旧页面 → Ctrl+F5 强制刷新
2. 检查 nginx 容器是否运行：`docker ps | findstr nginx`
3. 检查前端 dist 是否存在：迁移包内应包含 `frontend\dist\index.html`

### Q3：登录后提示 401 或页面反复刷新

这是旧版本的已知问题，新版本已修复（登录页改用公开接口获取站点标题）。如仍出现：
1. 确认使用的是最新代码（含 `useSiteTitle.ts` composable）
2. 清除浏览器 localStorage 后重新登录

### Q4：数据库导入报错"database is being accessed by other users"

导入脚本已自动处理连接终止，如仍报错：
1. 停止所有服务：`docker compose down`
2. 只启动 postgres：`docker compose up -d postgres`
3. 重新运行导入脚本

### Q5：想修改访问端口

编辑 `docker-compose.yml`，修改 `nginx` 服务的 ports 映射：

```yaml
nginx:
  ports:
    - "80:80"        # 改为 80 则访问 http://IP（无需输端口）
    # - "9000:80"    # 当前配置
```

修改后重启：`docker compose up -d nginx`

### Q6：如何在新服务器上修改前端

1. 安装 Node.js 18+：https://nodejs.org/
2. 进入 `frontend` 目录运行 `npm install`
3. 修改 `src/` 下的源码
4. 运行 `更新前端.bat` 重新构建
5. 构建产物自动复制到 nginx 容器

### Q7：迁移后上传的文件打不开

确认 uploads 目录权限：
1. 检查 `uploads` 文件夹是否存在且有内容
2. 确认 docker-compose.yml 中 backend 的 volume 映射正确：`./uploads:/app/uploads`

## 日常运维

### 启动系统
双击 `启动系统.bat`，等待所有容器启动（约 10-30s）。

### 停止系统
双击 `停止服务.bat`。

### 查看日志
```bash
docker compose logs -f backend      # 后端日志
docker compose logs -f nginx        # nginx 日志
docker compose logs -f postgres     # 数据库日志
```

### 开机自启
将 `启动系统.bat` 的快捷方式放入启动文件夹：
- 按 `Win+R`，输入 `shell:startup`
- 将快捷方式拖入打开的文件夹

### 数据定期备份
建议每周运行一次 `迁移-导出.ps1` 生成备份包，保留最近 4 周的备份。

---

## 回滚方案

如新服务器部署失败需回滚到原服务器：

1. 原服务器未做任何改动，直接 `启动系统.bat` 即可恢复
2. 如原服务器已停止，确认 Docker Desktop 运行后启动即可
3. 迁移过程中导出的 `迁移包` 可作为数据备份保留

**重要**：确认新服务器完全正常前，不要停止或删除原服务器的 Docker 容器和数据卷。

# 方案 D · 清新自然风（Fresh Natural）设计规范

> 设备信息动态管理系统 · 前端界面重构方案之一
> 设计理念：柔白背景 + 森林绿主色；圆润柔和、友好亲和；大圆角、柔和阴影、有机形态，像消费级健康/环保产品后台。

---

## 一、设计理念与适用场景

| 维度 | 说明 |
| --- | --- |
| 核心理念 | 以「亲和易用」为核心，降低一线工程师的使用心理门槛。借鉴 Headspace、Calm、Notion Health、消费级 SaaS 的圆润柔和美学，让工业系统也有温度。 |
| 视觉关键词 | 柔白、森林绿、圆润、大圆角、柔和阴影、有机、友好 |
| 适用场景 | 面向一线工程师、强调易用与温度的设备管家；工程师流动性大、IT 素养参差、需要降低学习成本的场景。 |
| 不适用场景 | 高密度数据监控（建议方案 B）；庄重合规档案（建议方案 C）。 |

### 2.3 状态色（自然色调）

| 语义 | Token | 色值 |
| --- | --- | --- |

| 警告 | `--state-warning` | `#D69E2E` |
| 错误 | `--state-error` | `#C53030` |

### 2.4 图表色（自然色系）

`--chart-1` `#2F855A` → `--chart-2` `#38B2AC` → `--chart-3` `#D69E2E` → `--chart-4` `#68A063` → `--chart-5` `#8B9D83`

**用色原则**：自然色系为主，绿 + 大地黄 + 青绿；状态色柔和不过曝；圆角与柔和阴影贯穿全部组件；禁止尖锐直角与高饱和荧光色。

## 四、组件库规范

### 4.1 卡片 `.card-natural`

- 背景 `#FFFFFF`，边框 `1px solid #E3EAE4`，圆角 `16px`（大圆角）
- 柔和阴影 `0 1px 2px rgba(31,42,36,0.04), 0 2px 8px rgba(31,42,36,0.04)`
- 内边距 `20px`

### 4.2 按钮

- 主按钮：`#2F855A` 底 + 白字，圆角 `12px`，hover `#276749`
- 次按钮：白底 + `#E3EAE4` 边框 + `#38503F` 字，圆角 `12px`
- 圆形图标按钮：`999px` 圆

### 4.3 表格

- 表头：`#F6F7F3` 底 + `#5C7268` 字
- 行高 `52px`（宽松舒适），行分隔 `1px solid #EAF0EA`
- 选中行 `#E3F6E9` 底

### 4.4 表单

- 输入框：`44px` 高（偏大易点），`#E3EAE4` 边框，圆角 `12px`，focus `#2F855A` 边框 + 浅绿环
- 标签：`13px` `#5C7268`

### 4.5 标签/徽章

- 叶片状柔和 `.pill-soft`：`999px` 圆角，`#E3F6E9` 底 + `#276749` 字
- 状态标签：成功绿、警告黄、错误红，均叶片状

### 4.6 侧边导航

- 宽 `220px`，`#FDFCF8` 底，激活项 `#E3F6E9` 底 + 圆角 `12px` + `#276749` 字

### 4.7 装饰元素

- `.pill-soft`：叶片状高亮
- 柔和阴影分层（卡片轻阴影 + 浮层重阴影）
- 有机圆角（不超 `16px`，避免过度）

## 五、间距规则

| Token | 值 | 用途 |
| --- | --- | --- |
| `--space-page` | `24px` | 页面外边距 |
| `--space-card` | `20px` | 卡片内边距 |
| 组件间距 | `16px / 20px` | 卡片间、表单项间 |
| 紧凑间距 | `8px` | 标签内、图标与文字 |
| 密度 `--density` | `3`（中等） | 信息密度档位 |

**网格**：12 栏，栏间距 `24px`，最大内容宽 `1440px`。表单页常用单栏居中（宽 `640px`），降低认知负荷。

## 六、交互设计说明

### 6.1 核心用户流程

1. **登录**：柔白登录页，工号 + 密码 → 「登录」(`btn-login`) → 仪表盘（友好卡片式）
2. **提交日志**：侧边「日志管理」(`nav-logs`) → 日志列表（卡片/行） → 行点击 (`row-log-1`) → 详情 → 「新建日志」(`btn-new-log`) → 表单（引导式）
3. **状态反馈**：提交成功用绿色叶片 Toast + 柔和动画

### 6.2 关键交互状态

| 节点 | 默认态 | 悬停态 | 激活/选中态 |
| --- | --- | --- | --- |
| 表格行 | 白底 | `#F6F7F3` 底 | `#E3F6E9` 底 + 圆角 |
| 主按钮 | `#2F855A` | `#276749` + 阴影增强 | 按下 `scale(0.98)` |
| 侧边导航项 | `#5C7268` 字 | `#38503F` 字 + `#F6F7F3` 底 + 圆角 | `#276749` 字 + `#E3F6E9` 底 + 圆角 |
| 输入框 | `#E3EAE4` 边框 | `#A8BCAB` 边框 | `#2F855A` 边框 + 浅绿环 |
| 叶片标签 | `#E3F6E9` 底 | — | — |

**转场**：页面切换 `250ms ease`（柔和）；卡片悬停 `150ms` 阴影上浮；Toast `300ms` 弹性缩放出现；整体节奏舒缓，避免突兀。

## 七、响应式布局

桌面优先（PC 端，最小 `1280px`）。断点：

- `≥1440px`：侧边栏 `240px` + 内容区满栏
- `1280–1440px`：侧边栏 `220px` + 内容区
- `<1280px`（降级）：侧边栏折叠 `64px` 图标栏，表单页保持单栏居中

8 种日志类型用叶片标签横向滚动筛选。

## 八、技术可行性（现有 Vue3 + Element Plus + ECharts 栈）

| 项 | 可行性 | 实现方式 |
| --- | --- | --- |
| 主题落地 | 高 | 引入 `element-plus-tokens-natural.css`，覆盖 `--el-*` 即可 |
| 大圆角 | 高 | `el-card/el-button/el-tag` 全局改 `border-radius` |
| 柔和阴影 | 高 | 覆盖 `--el-box-shadow*` 变量 |
| 叶片标签 | 高 | `el-tag { border-radius: 999px }` 全局覆盖 |
| ECharts 自然色 | 高 | `color` 读 `--chart-1..5` |
| 抗锯齿 | 高 | body `-webkit-font-smoothing: antialiased` |

**风险点**：① 大圆角在小尺寸组件（如密集表格操作列）可能拥挤，需局部降圆角；② 柔和阴影在低性能机上叠加多卡片可能卡顿，需控制阴影层数。其余零风险。该方案对现有 Element Plus 组件改动最小，落地最平滑。

# 方案 C · 复古怀旧风（Retro Nostalgic）设计规范

> 设备信息动态管理系统 · 前端界面重构方案之一
> 设计理念：暖纸背景 + 咖啡主色 + 衬线标题；文档式档案排版；方正卡片、实线边框、印章/工单感，像一本工业设备维修档案册。

---

## 一、设计理念与适用场景

| 维度 | 说明 |
| --- | --- |
| 核心理念 | 以「档案留痕」为核心，将设备维保日志塑造为可追溯的工业档案册。借鉴老式工单、设备维修台账、档案盒标签的纸质美学，强调记录的庄重与可信。 |
| 视觉关键词 | 暖纸、咖啡、衬线、方正、实线、印章、工单、档案 |
| 适用场景 | 注重记录留痕、档案属性强的设备维保系统；需要让一线工程师和管理员都感到「这是正式记录」的合规场景。 |
| 不适用场景 | 实时监控大屏（建议方案 B）；追求极简效率的高频录入（建议方案 A）。 |

## 二、色彩系统

### 2.1 品牌主色（咖啡棕，暖色单一色相）

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--primary-50` | `#F5EFE6` | 纸面浅底 |
| `--primary-100` | `#E8DCC6` | 浅底标签 |
| `--primary-300` | `#B08D6E` | 次级强调 |
| `--primary-500` | `#6F4E37` | **主色**（按钮、链接） |
| `--primary-700` | `#4A3322` | 按下态 |

### 2.2 暖中性阶（纸感）

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--paper-50` | `#FBF6EC` | 卡片背景 |
| `--paper-100` | `#F5EFE6` | 页面背景 |
| `--paper-200` | `#EDE3D2` | 次级背景 |
| `--paper-300` | `#D9C9B0` | 边框、分隔 |
| `--paper-500` | `#8A6F52` | 次要文本 |
| `--paper-700` | `#5C4633` | 正文主体 |
| `--ink-900` | `#3D2C1E` | 标题、墨色文本 |

### 2.3 状态色（泥土色系，复古调）

| 语义 | Token | 色值 | 色名 |
| --- | --- | --- | --- |
| 成功 | `--state-success` | `#4B6B3B` | 橄榄绿 |
| 警告 | `--state-warning` | `#B7410E` | 赭石/铁锈 |
| 错误 | `--state-error` | `#8B2C1B` | 暗砖红 |
| 信息 | `--state-info` | `#6F4E37` | 咖啡 |

### 2.4 图表色（大地色系）

`--chart-1` `#6F4E37` → `--chart-2` `#B7410E` → `--chart-3` `#4B6B3B` → `--chart-4` `#C9A227` → `--chart-5` `#8A6F52`

**用色原则**：全系统暖色基调，大地色系；状态色用泥土色而非鲜艳色，保持复古统一；图表用大地色系避免突兀；禁止冷色与高饱和色。

## 三、字体规范

### 3.1 字体族

| Token | 字体栈 | 用途 |
| --- | --- | --- |
| `--font-serif` | `'Noto Serif SC', 'Source Han Serif SC', Georgia, 'Times New Roman', serif` | **标题**（档案册感） |
| `--font-sans` | `'Noto Sans SC', -apple-system, 'Segoe UI', system-ui, sans-serif` | 正文、界面 |
| `--font-mono` | `'Courier New', 'SFMono-Regular', monospace` | 编号、数字（工单号） |

### 3.2 字号阶（px）

| 用途 | 字号 | 行高 | 字重 | 字体 |
| --- | --- | --- | --- | --- |
| 档案标题 H1 | 26 | 34 | 900 | 衬线 |
| 区块标题 H2 | 19 | 28 | 700 | 衬线 |
| 卡片标题 H3 | 16 | 24 | 700 | 衬线 |
| 正文 Body | 14 | 24 | 400 | 无衬线 |
| 辅助文本 | 13 | 20 | 400 | 无衬线 |
| 工单编号 | 13 | 18 | 500 | 等宽 |
| KPI 数字 | 30 | 38 | 700 | 衬线 |

### 3.3 字距

标题 `letter-spacing: 0.02em` 略松；印章 `letter-spacing: 0.1em` 强调庄重；编号等宽 `tabular-nums`。

## 四、组件库规范

### 4.1 卡片 `.card-retro`

- 背景 `#FBF6EC`，边框 `1px solid #D9C9B0`，圆角 `2px`
- 顶部柔棕渐变 `linear-gradient(180deg, rgba(176,141,110,0.04), transparent 120px)`
- 内边距 `22px`（中等偏松，文档式）

### 4.2 按钮

- 主按钮：`#6F4E37` 底 + `#FBF6EC` 字，圆角 `2px`，hover `#4A3322`
- 次按钮：`#FBF6EC` 底 + `#D9C9B0` 边框 + `#5C4633` 字
- 印章按钮（特殊）：`.stamp` 类，砖红双线边框 + `rotate(-6deg)`

### 4.3 表格

- 表头：`#EDE3D2` 底 + 衬线 `#5C4633` 字
- 行高 `44px`，行分隔 `1px solid #EDE3D2`
- 选中行 `#F5EFE6` 底

### 4.4 表单

- 输入框：`40px` 高，`#D9C9B0` 边框，圆角 `2px`，focus `#6F4E37` 边框
- 标签：`13px` 衬线 `#8A6F52`
- 工单编号字段：等宽字体，带「№」前缀

### 4.5 标签/徽章

- 方正徽章：`#EDE3D2` 底 + `#5C4633` 字，圆角 `2px`
- 印章标签 `.stamp`：状态标记用，如「已审核」「已报废」

### 4.6 侧边导航

- 宽 `220px`，`#F5EFE6` 底，激活项 `#E8DCC6` 底 + 左侧 `2px` 咖啡色条 + 衬线标题

### 4.7 装饰元素

- `.font-display`：衬线标题类
- `.stamp`：印章（双线边框、旋转、字距加宽）
- 卡片顶部柔棕渐变模拟纸张光影

## 五、间距规则

| Token | 值 | 用途 |
| --- | --- | --- |
| `--space-page` | `28px` | 页面外边距（文档式偏松） |
| `--space-card` | `22px` | 卡片内边距 |
| 组件间距 | `16px / 20px` | 卡片间、表单项间 |
| 紧凑间距 | `8px` | 标签内、图标与文字 |
| 密度 `--density` | `2`（中等偏松） | 信息密度档位 |

**网格**：12 栏，栏间距 `24px`，最大内容宽 `1400px`。详情页常用双栏（左主信息 + 右档案元数据）。

## 六、交互设计说明

### 6.1 核心用户流程

1. **登录**：纸感登录页，工号 + 密码 → 「登录」(`btn-login`) → 仪表盘（档案目录式）
2. **查阅档案**：侧边「日志管理」(`nav-logs`) → 日志列表（工单台账） → 行点击 (`row-log-1`) → 日志详情（档案卡）
3. **登记新工单**：详情页「新建日志」(`btn-new-log`) → 表单（工单填写式，带编号预览）

### 6.2 关键交互状态

| 节点 | 默认态 | 悬停态 | 激活/选中态 |
| --- | --- | --- | --- |
| 表格行 | `#FBF6EC` 底 | `#F5EFE6` 底 | `#E8DCC6` 底 + 左色条 |
| 主按钮 | `#6F4E37` | `#4A3322` | 按下 `scale(0.98)` |
| 侧边导航项 | `#8A6F52` 字 | `#5C4633` 字 + `#EDE3D2` 底 | `#4A3322` 字 + `#E8DCC6` 底 |
| 输入框 | `#D9C9B0` 边框 | `#B08D6E` 边框 | `#6F4E37` 边框 |
| 印章 | `opacity 0.85` | — | 审核通过后浮现 `scale` 动画 |

**转场**：页面切换 `200ms ease`（纸页翻动感）；印章出现用 `300ms` 缩放 + 旋转定格；浮层用柔棕阴影 `200ms` 淡入。

## 七、响应式布局

桌面优先（PC 端，最小 `1280px`）。断点：

- `≥1440px`：侧边栏 `240px` + 详情页双栏
- `1280–1440px`：侧边栏 `220px` + 详情页单栏
- `<1280px`（降级）：侧边栏折叠 `64px`，详情页单栏

档案台账表格保持完整列，不随屏宽折叠。

## 八、技术可行性（现有 Vue3 + Element Plus + ECharts 栈）

| 项 | 可行性 | 实现方式 |
| --- | --- | --- |
| 主题落地 | 高 | 引入 `element-plus-tokens-retro.css`，覆盖 `--el-*` 即可 |
| 衬线标题 | 高 | `.el-card__header/.el-dialog__title` 全局改 `font-family: Noto Serif SC` |
| 方正卡片 | 高 | `el-card { border-radius: 2px }` 全局覆盖 |
| 印章效果 | 中 | `.stamp` 类自定义 `border/transform/letter-spacing`；CSS 即可，无 JS |
| 纸面渐变 | 高 | `el-card::before` 或 `background-image` |
| ECharts 大地色 | 高 | `color` 读 `--chart-1..5` |
| 等宽编号 | 高 | `.numeric` 类 + Courier New |

**风险点**：① 衬线字体在低 DPI 屏可能发虚，需测试；② 内网需本地化 Noto Serif SC；③ 印章旋转需注意表格内对齐。其余零风险。

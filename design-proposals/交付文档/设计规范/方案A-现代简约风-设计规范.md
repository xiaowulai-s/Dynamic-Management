# 方案 A · 现代简约风（Modern Minimal）设计规范

> 设备信息动态管理系统 · 前端界面重构方案之一
> 设计理念：克制、理性、留白；单一品牌主色 + 中性灰阶；细线分隔、无阴影静态卡片、紧凑理性的信息层级。

---

## 一、设计理念与适用场景

| 维度 | 说明 |
| --- | --- |
| 核心理念 | 以「效率优先」为原则，去除一切装饰性元素，让信息本身成为主角。借鉴 Linear、Notion、Vercel 等现代 SaaS 后台的克制美学。 |
| 视觉关键词 | 克制、理性、留白、细线、紧凑、信息密度 |
| 适用场景 | 注重效率与信息密度的企业内网后台；一线工程师高频提交日志、管理员批量审核的日常作业场景。 |
| 不适用场景 | 需要强情感连接或品牌温度的对外展示；实时监控大屏（建议用方案 B）。 |

## 二、色彩系统

### 2.1 品牌主色（石板蓝，单一色相 + 明度阶）

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--primary-50` | `#EFF6FF` | 主色最浅底（hover/选中行底） |
| `--primary-100` | `#DBEAFE` | 浅底标签、徽章背景 |
| `--primary-500` | `#3B82F6` | 次级强调 |
| `--primary-600` | `#2563EB` | **主色**（按钮、链接、激活态） |
| `--primary-700` | `#1D4ED8` | 主色按下态 |

### 2.2 中性灰阶

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--gray-50` | `#F8FAFC` | 页面背景 |
| `--gray-100` | `#F1F5F9` | 次级背景、分隔区 |
| `--gray-200` | `#E2E8F0` | 边框、分隔线 |
| `--gray-300` | `#CBD5E1` | 禁用边框 |
| `--gray-400` | `#94A3B8` | 占位符文本 |
| `--gray-500` | `#64748B` | 次要文本 |
| `--gray-600` | `#475569` | 正文辅助 |
| `--gray-700` | `#334155` | 正文主体 |
| `--gray-900` | `#0F172A` | 标题、主文本 |

### 2.3 语义色

| 语义 | Token | 色值 |
| --- | --- | --- |
| 成功 | `--state-success` | `#16A34A` |
| 警告 | `--state-warning` | `#D97706` |
| 错误 | `--state-error` | `#DC2626` |
| 信息 | `--state-info` | `#2563EB` |

### 2.4 图表色（主色阶 + 中性，单色相递进）

`--chart-1` `#2563EB` → `--chart-2` `#60A5FA` → `--chart-3` `#93C5FD` → `--chart-4` `#64748B` → `--chart-5` `#CBD5E1`

**用色原则**：全系统仅一个品牌色相（蓝），通过明度阶构建层次；状态色仅用于状态标识，不参与装饰；禁止使用渐变、发光、阴影作为装饰。

## 三、字体规范

### 3.1 字体族

| Token | 字体栈 | 用途 |
| --- | --- | --- |
| `--font-sans` | `'Inter', 'Noto Sans SC', -apple-system, system-ui, sans-serif` | 正文、界面 |
| `--font-mono` | `'JetBrains Mono', 'SFMono-Regular', Consolas, monospace` | 数字、编号、代码 |

### 3.2 字号阶（px）

| 用途 | 字号 | 行高 | 字重 |
| --- | --- | --- | --- |
| 页面标题 H1 | 28 | 36 | 700 |
| 区块标题 H2 | 20 | 28 | 600 |
| 卡片标题 H3 | 16 | 24 | 600 |
| 正文 Body | 14 | 22 | 400 |
| 辅助文本 | 13 | 20 | 400 |
| 标签/编号 | 12 | 18 | 500 |
| KPI 数字 | 32 | 40 | 700（等宽） |

### 3.3 字距

正文 `letter-spacing: -0.005em`；数字等宽 `font-variant-numeric: tabular-nums`。

## 四、组件库规范

### 4.1 卡片 `.card-minimal`

- 背景 `#FFFFFF`，边框 `1px solid #E2E8F0`，圆角 `8px`
- **无投影**（`box-shadow: none`），层次靠边框与留白区分
- 内边距 `20px`

### 4.2 按钮

- 主按钮：`#2563EB` 底 + 白字，圆角 `8px`，hover `#1D4ED8`
- 次按钮：白底 + `#E2E8F0` 边框 + `#334155` 字
- 文字按钮：无边框，`#2563EB` 字

### 4.3 表格

- 表头：`#F8FAFC` 底 + `#64748B` 字 + `12px` 字号
- 行高 `48px`，行分隔线 `1px solid #F1F5F9`
- 选中行 `#EFF6FF` 底

### 4.4 表单

- 输入框：`40px` 高，`#E2E8F0` 边框，圆角 `8px`，focus 边框 `#2563EB`
- 标签：`13px` `#64748B`

### 4.5 标签/徽章

- 浅底圆角徽章：`#EFF6FF` 底 + `#1D4ED8` 字，圆角 `4px`

### 4.6 侧边导航

- 宽 `220px`，白底，激活项 `#EFF6FF` 底 + 左侧 `2px` `#2563EB` 色条 + `#1D4ED8` 字

## 五、间距规则

| Token | 值 | 用途 |
| --- | --- | --- |
| `--space-page` | `24px` | 页面外边距 |
| `--space-card` | `20px` | 卡片内边距 |
| 组件间距 | `12px / 16px` | 卡片间、表单项间 |
| 紧凑间距 | `8px` | 标签内、图标与文字 |
| 密度 `--density` | `3`（中等偏紧） | 信息密度档位 |

**网格**：12 栏，栏间距 `24px`，最大内容宽 `1440px`。

## 六、交互设计说明

### 6.1 核心用户流程

1. **登录**：输入工号 + 密码 → 点击「登录」(`btn-login`) → 跳转仪表盘
2. **提交日志**：仪表盘 → 侧边「日志管理」(`nav-logs`) → 日志列表 → 点击某行 (`row-log-1`) → 日志详情 → 「新建日志」(`btn-new-log`) → 表单填写 → 提交
3. **审核日志**：日志列表 → 详情页查看 → 通过/驳回

### 6.2 关键交互状态

| 节点 | 默认态 | 悬停态 | 激活/选中态 |
| --- | --- | --- | --- |
| 表格行 | 白底 | `#F8FAFC` 底 | `#EFF6FF` 底 + 左色条 |
| 主按钮 | `#2563EB` | `#1D4ED8` | 按下 `scale(0.98)` |
| 侧边导航项 | `#64748B` 字 | `#334155` 字 + `#F8FAFC` 底 | `#1D4ED8` 字 + `#EFF6FF` 底 |
| 输入框 | `#E2E8F0` 边框 | `#CBD5E1` 边框 | `#2563EB` 边框 + 浅蓝环 |

**转场**：页面切换无动画（效率优先）；仅浮层/抽屉使用 `150ms ease` 淡入。

## 七、响应式布局

桌面优先（PC 端局域网，最小 `1280px`）。断点：

- `≥1440px`：侧边栏 `240px` + 内容区满栏
- `1280–1440px`：侧边栏 `220px` + 内容区
- `<1280px`（降级）：侧边栏折叠为图标栏 `64px`

8 种日志类型在列表用标签筛选，不随屏宽变化布局。

## 八、技术可行性（现有 Vue3 + Element Plus + ECharts 栈）

| 项 | 可行性 | 实现方式 |
| --- | --- | --- |
| 主题落地 | 高 | 直接引入 `element-plus-tokens-minimal.css`，覆盖 `--el-*` 变量即可，无需改组件结构 |
| 卡片无投影 | 高 | `el-card { box-shadow: none }` 全局覆盖 |
| 等宽数字 | 高 | KPI 用 `.numeric` 类或 `el-statistic` + 等宽字体 |
| 图表配色 | 高 | ECharts `color` 读取 `--chart-1..5`（`getComputedStyle`） |
| 细线表格 | 高 | Element Plus 表格默认即细线，调 `--el-border-color` |
| 无新增依赖 | 高 | 字体走 Google Fonts CDN（内网若禁外网则改用本地 Inter/Noto Sans SC） |

**风险点**：内网若无外网，Google Fonts 需替换为本地字体包；其余零风险。

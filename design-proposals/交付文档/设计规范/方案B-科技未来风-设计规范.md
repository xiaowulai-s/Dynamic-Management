# 方案 B · 科技感未来风（Tech Futuristic · Dark）设计规范

> 设备信息动态管理系统 · 前端界面重构方案之一
> 设计理念：深空背景 + 青蓝主色 + 等宽数字；高密度数据监控感；细发光描边、技术网格、扫描线，像指挥中心的设备监控大屏。

---

## 一、设计理念与适用场景

| 维度 | 说明 |
| --- | --- |
| 核心理念 | 以「实时感知」为核心，营造指挥中心式的高密度数据监控氛围。借鉴 Grafana、Datadog、Vercel Observability 等运维监控工具的暗色大屏美学。 |
| 视觉关键词 | 深空、青蓝、发光描边、网格、扫描线、等宽数字、高密度 |
| 适用场景 | 实时设备监控、数据大屏、运维中心；需要一眼扫读海量设备状态的场景。 |
| 不适用场景 | 长时间表单录入（深底易疲劳）；档案留痕属性强的场景（建议用方案 C）。 |

### 2.4 图表色（青/蓝/绿/紫/灰，数据屏配色）

`--chart-1` `#22D3EE` → `--chart-2` `#38BDF8` → `--chart-3` `#34D399` → `--chart-4` `#A78BFA` → `--chart-5` `#64748B`

**用色原则**：深底 + 单一青蓝主色 + 多色图表（数据区分需多色）；发光仅用于浮层、主按钮、强调描边，静态卡片用细描边不发光；网格背景透明度极低（`0.06`）仅作氛围。

## 三、字体规范

### 3.1 字体族

| Token | 字体栈 | 用途 |
| --- | --- | --- |
| `--font-sans` | `'Space Grotesk', 'Noto Sans SC', system-ui, sans-serif` | 标题、界面 |
| `--font-mono` | `'JetBrains Mono', 'SFMono-Regular', Consolas, monospace` | 数字、编号、代码 |

### 3.2 字号阶（px）

| 用途 | 字号 | 行高 | 字重 |
| --- | --- | --- | --- |
| 大屏标题 H1 | 28 | 36 | 700（几何无衬线） |
| 区块标题 H2 | 20 | 28 | 600 |
| 卡片标题 H3 | 15 | 22 | 600 |
| 正文 Body | 14 | 22 | 400 |
| 辅助文本 | 12 | 18 | 400 |
| KPI 数字 | 32 | 40 | 700（等宽，字距 `0.02em`） |

### 3.3 字距

正文 `letter-spacing: 0`；数字 `letter-spacing: 0.02em` 增强技术感；标题用 Space Grotesk 几何字形。

## 四、组件库规范

### 4.1 卡片 `.card-tech`

- 背景 `#131C2E`，边框 `1px solid #1E293B`，圆角 `4px`
- `::before` 伪元素：顶部青色高光渐变 `linear-gradient(180deg, rgba(34,211,238,0.06), transparent 40%)`
- 内边距 `16px`（高密度）

### 4.2 按钮

- 主按钮：`#22D3EE` 底 + 深字 `#04111A`，发光描边 `--glow-primary`
- 次按钮：透明底 + `#1E293B` 边框 + `#CBD5E1` 字
- 图标按钮：`#1E293B` 底 + 青色图标

### 4.3 表格

- 表头：`#0F172A` 底 + `#94A3B8` 字 + 等宽
- 行高 `40px`（紧凑），行分隔 `1px solid #1E293B`
- 悬停行 `#1E293B` 底

### 4.4 表单

- 输入框：`36px` 高，`#1E293B` 底 + 同色边框，圆角 `4px`，focus 青色发光描边
- 标签：`12px` `#94A3B8` 大写感

### 4.5 标签/徽章

- 发光徽章：`#0E2A33` 底 + `#22D3EE` 字 + 青色细边框

### 4.6 侧边导航

- 宽 `200px`，`#0F172A` 底，激活项青色左色条 + `#22D3EE` 字 + 微发光

### 4.7 装饰元素

- `.scanline`：扫描线纹理（`repeating-linear-gradient`，3px 间隔，青色 `0.04` 透明）
- 网格背景：body `linear-gradient` 双向网格，`32px` 格，`rgba(148,163,184,0.06)`

## 五、间距规则

| Token | 值 | 用途 |
| --- | --- | --- |
| `--space-page` | `20px` | 页面外边距（紧凑） |
| `--space-card` | `16px` | 卡片内边距 |
| 组件间距 | `8px / 12px` | 卡片间、表单项间 |
| 紧凑间距 | `4px` | 标签内、图标与文字 |
| 密度 `--density` | `5`（最高密度） | 信息密度档位 |

**网格**：12 栏，栏间距 `20px`，最大内容宽 `1440px`。KPI 区常用 4–6 栏并排（高密度）。

## 六、交互设计说明

### 6.1 核心用户流程

1. **登录**：深色登录页，工号 + 密码 → 「登录」(`btn-login`) → 仪表盘（大屏式）
2. **监控**：仪表盘实时展示设备状态分布（环形图）、日志趋势（折线）、类型分布（柱状）
3. **日志流转**：侧边「日志管理」(`nav-logs`) → 列表 → 行点击 (`row-log-1`) → 详情 → 「新建日志」(`btn-new-log`) → 表单

### 6.2 关键交互状态

| 节点 | 默认态 | 悬停态 | 激活/选中态 |
| --- | --- | --- | --- |
| 表格行 | `#131C2E` 底 | `#1E293B` 底 | `#0E2A33` 底 + 青色左条 |
| 主按钮 | 青底 + 发光 | 增强发光 `0.25` | 按下 `scale(0.98)` |
| 侧边导航项 | `#94A3B8` 字 | `#CBD5E1` 字 | `#22D3EE` 字 + 青色左条 + 微发光 |
| 输入框 | `#1E293B` 边框 | `#334155` 边框 | 青色发光描边 |
| KPI 卡片 | 细描边 | 青色高光增强 | — |

**转场**：数据更新用 `300ms ease` 数字滚动；浮层 `200ms` 淡入 + 发光描边动画；扫描线静态装饰。

## 七、响应式布局

桌面优先（PC 端，最小 `1280px`）。断点：

- `≥1440px`：侧边栏 `220px` + KPI 6 栏并排
- `1280–1440px`：侧边栏 `200px` + KPI 4 栏
- `<1280px`（降级）：侧边栏折叠 `64px` 图标栏，KPI 降为 3 栏

监控大屏建议固定 `1920×1080` 投屏场景。

## 八、技术可行性（现有 Vue3 + Element Plus + ECharts 栈）

| 项 | 可行性 | 实现方式 |
| --- | --- | --- |
| 深色主题 | 高 | 引入 `element-plus-tokens-tech.css` + `html.dark` 类；Element Plus 原生支持 dark |
| 发光描边 | 高 | `box-shadow` + `--glow-primary` 变量；按钮/输入框全局覆盖 |
| 网格背景 | 高 | body `background-image` 双向 `linear-gradient` |
| 扫描线 | 高 | `.scanline` 类 `repeating-linear-gradient` |
| ECharts 暗色 | 高 | `echarts.init(dom, 'dark')` + 自定义 `color` 读 `--chart-*` |
| 等宽数字 | 高 | `.numeric` 类 + JetBrains Mono |

**风险点**：① 深底长时间表单录入易疲劳，建议表单页可切浅底或增大对比；② 内网需本地化 Space Grotesk/JetBrains Mono 字体包；③ 发光描边在低性能机上需控制使用频率（仅主按钮/激活态）。其余零风险。

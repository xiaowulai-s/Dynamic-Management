越权防护规则 后端强制校验 · 前端控制仅为体验优化

  1. 1
    用户列表过滤
    admin 登录时，GET /auth/users 在 SQL 层面 WHERE role != 'super_admin'，即使绕过前端也无法获取超管信息。
    SQL 过滤
  2. 2
    用户操作边界
    admin 对 super_admin 的增 / 改 / 删操作在后端校验角色后直接返回 403 Forbidden。
    403
  3. 3
    自删防护
    任何角色都不能删除自己，后端校验 current_user.id == user_id 时返回 400。
    400
  4. 4
    日志编辑状态
    后端校验 log.status in ('pending','rejected')，已通过（approved）日志任何角色都不可修改。
    状态锁
  5. 5
    日志删除
    普通用户即使构造 DELETE 请求，后端在角色校验阶段即返回 403 Forbidden。
    403
